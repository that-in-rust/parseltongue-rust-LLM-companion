  Key Components Needed

  1. Git Cloner Service

  User input: github.com/facebook/react
     │
     ▼
  Shallow clone: git clone --depth 1 --branch main
     │
     ▼
  Temp directory: /tmp/jobs/{job_id}/repo

  2. Job Queue (for async processing)

  - Large repos take time (React = 2000+ files)
  - Queue jobs, return job_id immediately
  - WebSocket/SSE for progress updates

  3. Cache Strategy

  Cache key: SHA256(repo_url + default_branch + last_commit_sha)
  TTL: 24 hours (or webhook-based invalidation)

  Hit?  → Return cached graph instantly
  Miss? → Queue analysis job

  4. Cost Considerations

  | Component  | Challenge                         | Solution
     |
  |------------|-----------------------------------|------------------------
  ---|
  | Clone time | Large repos slow                  | Shallow clone (--depth 
  1) |
  | Disk space | Repos accumulate                  | Temp files + cleanup
     |
  | CPU        | Parsing is intensive              | Worker pool + queue
     |
  | Storage    | Graph DBs grow                    | Compressed JSON + TTL
     |
  | Abuse      | Someone indexes linux kernel 100x | Rate limiting + cache
     |

  5. Rate Limiting

  - Per-IP: 10 requests/hour
  - Per-repo: Cache for 24h
  - Repo size limit: < 500MB, < 10K files

  Two Deployment Options

  Option A: Serverless (Lower cost, slower)

  Vercel/Cloudflare → Lambda/Workers → S3
  - Cold start penalty
  - 15 min timeout risk for large repos
  - ~$0 for light usage

  Option B: Always-on Workers (Faster, fixed cost)

  Fly.io / Railway → Redis Queue → Workers
  - No cold start
  - Handles large repos
  - ~$20-50/month

  Minimum Viable Version

  1. Frontend: Single page with URL input + graph viewer
  2. Backend:  Single server with parseltongue binary
  3. Storage:  Local disk with LRU eviction
  4. Cache:    In-memory HashMap (repo_url → graph_json)

  This is NOT the MCP use case - this is HTTP API for web service,
  completely separate from the local STDIO MCP server we discussed for
  Claude Code integration.

  // ~50 lines of JS, no dependencies
  const patterns = {
    js: {
      import: /import\s+.*\s+from\s+['"](.+)['"]/g,
      require: /require\(['"](.+)['"]\)/g,
    },
    python: {
      import: /^import\s+(\S+)/gm,
      from: /^from\s+(\S+)\s+import/gm,
    },
    rust: {
      use: /^use\s+(\S+)/gm,
      mod: /^mod\s+(\w+)/gm,
    }
  };

  function extractDeps(code, lang) {
    const deps = [];
    for (const [name, regex] of Object.entries(patterns[lang])) {
      let match;
      while ((match = regex.exec(code)) !== null) {
        deps.push({ type: name, target: match[1] });
      }
    }
    return deps;
  }

  self.addEventListener('install', (event) => {

  // Main app - load WASM from cache
  async function loadParseltongue() {
    const cache = await caches.open('parseltongue-v1');
    const response = await cache.match('/parseltongue.wasm');
    const wasm = await WebAssembly.instantiateStreaming(response);
    return wasm.instance.exports;
  }

  // Store analysis results
  async function cacheAnalysis(repoUrl, graph) {
    const db = await openDB('parseltongue', 1);
    await db.put('analyses', { url: repoUrl, graph, timestamp: Date.now()
