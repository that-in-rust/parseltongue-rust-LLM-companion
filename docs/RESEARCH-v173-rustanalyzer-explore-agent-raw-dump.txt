### 14. COMPLETE WORKFLOW EXAMPLE

```rust
// Load codebase
let (mut db, _vfs, _pm) = load_workspace_at(
    ".",
    &CargoConfig::default(),
    &LoadCargoConfig {
        load_out_dirs_from_check: true,
        with_proc_macro_server: ProcMacroServerChoice::Sysroot,
        prefill_caches: true,
        proc_macro_processes: 0,
    },
    &|msg| println!("{msg}"),
)?;

// Create analysis snapshot
let analysis = db.analysis();

// Get crate info
let krate = Crate::all(&db).first().unwrap();
let root = krate.root_module(&db);

// Find all functions
for module in krate.modules(&db) {
    for decl in module.declarations(&db) {
        if let ModuleDef::Function(func) = decl {
            let name = func.name(&db);
            let ty = func.ty(&db);
            let params = func.params_without_self(&db);
            let ret_ty = func.ret_type(&db);
            println!("fn {}: {:?}", name, ret_ty);
        }
    }
}

// Find trait impls for a type
for impl_def in Impl::all_for_type(&db, some_type) {
    if let Some(trait_) = impl_def.trait_(&db) {
        println!("impl {} for ...", trait_.name(&db));
        for item in impl_def.items(&db) {
            println!("  - {}", item.name(&db));
        }
    }
}

// Method resolution
let scope = SemanticsScope::new(&analysis, position);
some_type.iterate_method_candidates(&db, &scope, None, |method| {
    println!("Available method: {}", method.name(&db));
    None
});
```

### 15. KEY ARCHITECTURAL POINTS FOR PT04

1. **Thread Safety**: `RootDatabase` is `Send + Sync` (uses Arc internally)
2. **Cancellation**: Use `Cancellable<T>` return type for long-running operations
3. **Lifetime Binding**: Many types require `'db` lifetime binding to database
4. **Query System**: Salsa-based incremental computation with automatic caching
5. **No Mutable Iteration**: Queries return snapshots; modifications create new snapshots
6. **Macro Support**: Full macro expansion with proc-macro support
7. **Edition-aware**: Handles Rust 2015/2018/2021 with `EditionedFileId`
8. **Trait Coherence**: Orphan rule checking and trait resolution support

### SUMMARY

rust-analyzer exposes a comprehensive semantic analysis API through three main layers:

1. **IDE Layer** (`ide` crate) - High-level queries returning user-friendly results
2. **HIR Layer** (`hir` crate) - Type definitions and semantic information
3. **Database Layer** (`ide_db`, `hir_ty`, `hir_def`) - Query infrastructure

For pt04, the key APIs are:
- `AnalysisHost`/`Analysis` for queries
- `Semantics` for source-to-def mapping
- `Type<'db>` for type analysis
- `Impl`/`Trait`/`Function` for code structure
- `load_workspace_*` for loading Cargo projects

All types are fully resolved with proper generics, lifetimes, and trait bounds -- exactly what Parseltongue needs for semantic analysis.
