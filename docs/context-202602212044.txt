## What is Delegate

Delegate is a Python-based multi-agent coding orchestrator by Nikhil Garg. It creates a persistent team of AI agents (Claude Code instances) on your local machine that:
- Plan tasks (manager agent with Opus)
- Write code in parallel (5 engineer agents with Sonnet)
- Review each other's work (peer agent review built into workflow)
- Merge to your local repo (git worktrees, rebase, fast-forward)
- Run via real-time browser UI (http://localhost:3548)

## 9 System Invariants (from Delegate's invariants.md)

1. Filesystem Isolation — agents only write to designated directories
2. Sandbox Layers — 5 independent security layers
3. Git State Management — agents can't modify branch topology
4. UUID Identity — teams/members identified by UUID internally
5. Dependency Ordering — worktree created only after deps resolved
6. Daemon Singleton — fcntl.flock() ensures only one daemon
7. MCP Tool Boundary — agents access DB through in-process MCP tools
8. Network Isolation — domain allowlist in protected/network.yaml
9. Migration Safety — numbered SQL migrations with backup + verify

## Conclusion

Delegate is an IMPLEMENTATION of the patterns in notes01-agent.md:
- Pattern 6: Isolate Context → sub-agents
- Ralph Loop → workflow stages with guards
- Gas Town → multi-agent + merge queue
- Pattern 4: Offload Context → context.md per agent
- Pattern 7: Evolve Context → memory journals

The tradeoff: orchestration sophistication vs context depth. For Rust project with very specific engineering principles (4WNC, S06, TDD-First), our custom agents have MORE wisdom but Delegate has BETTER coordination.
